## Problem Analysis
The container uses mirror+file configuration where apt-mirrors.txt contains ONLY AMD64-compatible mirrors.
When ARM64 architecture is added, apt tries to fetch ARM64 packages from these AMD64-only mirrors, causing 404 errors.

## Solution Required
Replace the mirror+file configuration with architecture-specific entries:
- AMD64 entries should use archive.ubuntu.com and security.ubuntu.com
- ARM64 entries should use ports.ubuntu.com/ubuntu-ports
- Each entry must have Architectures: field to prevent cross-contamination
